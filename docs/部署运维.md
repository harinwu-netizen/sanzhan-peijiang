## 1. 架构总览

三战配将项目采用**纯静态 + Serverless** 架构,首选部署平台是 [Vercel](https://vercel.com/)(Next.js 官方维护者做的,零配置集成)。

```
                ┌──────────────────────┐
                │    用户浏览器         │
                │  Chrome/Safari/微信   │
                └──────────┬───────────┘
                           │ HTTPS
                           ▼
                ┌──────────────────────┐
                │   Cloudflare CDN     │  (可选,接入自定义域名时)
                └──────────┬───────────┘
                           │
                           ▼
                ┌──────────────────────┐
                │   Vercel Edge        │
                │  (Next.js App Router) │
                │                      │
                │  • 静态页面 SSG/ISR  │
                │  • API Routes        │
                │  • 静态资源 CDN      │
                └──────────┬───────────┘
                           │
                ┌──────────┴───────────┐
                ▼                      ▼
    ┌──────────────────┐    ┌──────────────────┐
    │  GitHub Repo     │    │  Vercel KV/Blob  │  (M2+ 启用)
    │  data/*.json     │    │  (用户配将分享)   │
    │  (在仓库里)      │    │                  │
    └──────────────────┘    └──────────────────┘
```

MVP 阶段数据全在仓库里(`data/*.json`),**没有外部数据库**,部署即数据同步。M2 之后才考虑加 Vercel KV(用户配将分享数据)。

## 2. Vercel 部署(推荐)

### 2.1 前置准备

- 一个 GitHub 账号
- 仓库代码已推送到 GitHub(`main` 分支)
- 一个 Vercel 账号(可用 GitHub 登录,免费 Hobby 套餐够用)

### 2.2 导入项目

1. 登录 [Vercel Dashboard](https://vercel.com/dashboard)
2. 点击 **"Add New → Project"**
3. 在 **"Import Git Repository"** 中选择本仓库(需先授权 Vercel 访问你的 GitHub)
4. 进入配置页:

   | 配置项 | 值 |
   | --- | --- |
   | **Project Name** | `sanzhan-peijiang`(会变成 `sanzhan-peijiang.vercel.app` 默认域名) |
   | **Framework Preset** | Next.js(自动检测) |
   | **Root Directory** | `./`(留空) |
   | **Build Command** | `pnpm build`(Vercel 会自动用 pnpm) |
   | **Output Directory** | `.next`(默认) |
   | **Install Command** | `pnpm install`(默认) |
   | **Node.js Version** | 20.x(在 `package.json` 加 `"engines": { "node": "20.x" }` 可锁定) |

5. **Environment Variables**(首次可不填,MVP 无需环境变量)
6. 点击 **"Deploy"**,等待约 1-3 分钟

### 2.3 部署成功

部署完成后会得到一个默认域名:
```
https://sanzhan-peijiang.vercel.app
```

打开即可访问。每次推送到 `main` 都会触发自动部署(Production),其他分支会部署到 Preview URL。

### 2.4 部署配置参考(`vercel.json`)

可选,放项目根目录:

```json
{
  "buildCommand": "pnpm build",
  "framework": "nextjs",
  "regions": ["hkg1", "sin1"],
  "github": {
    "silent": false
  }
}
```

- `regions: ["hkg1", "sin1"]` — 香港 + 新加坡,对中国大陆和东南亚用户友好
- 默认 Vercel 节点在美国(可能慢),改成亚洲节点能显著降低首屏延迟

### 2.5 包管理器

Vercel 会自动检测 `pnpm-lock.yaml`,使用 pnpm。如果没自动检测,在 `Settings → General → Package Manager` 手动选 `pnpm`。

## 3. 自定义域名

### 3.1 购买域名

推荐域名注册商(按中国大陆访问友好度排序):

| 注册商 | 优点 | 缺点 |
| --- | --- | --- |
| [Cloudflare Registrar](https://www.cloudflare.com/products/registrar/) | 成本价,免费 WHOIS 隐私 | 流程稍慢 |
| [NameSilo](https://www.namesilo.com/) | 价格便宜,免费 WHOIS 隐私 | UI 旧 |
| [阿里云(万网)](https://wanwang.aliyun.com/) | 国内备案方便 | 需要实名 |
| [腾讯云 DNSPod](https://www.dnspod.cn/) | 国内解析快 | 需要实名 |

建议域名:`sanzhanpeijiang.com` / `sanzhan-pj.com` / `peijiang.pro` / `3zhan.tools` 等。

### 3.2 在 Vercel 添加域名

1. 进入项目 → **Settings → Domains**
2. 输入要绑定的域名(如 `sanzhanpeijiang.com`)
3. Vercel 会提示添加 DNS 记录

**情况 A:域名在 Cloudflare(推荐)**

Cloudflare 和 Vercel 配合最好,直接 CNAME 即可:

1. 在 Cloudflare DNS 添加:
   ```
   类型:CNAME
   名称:@(根域名) 或 www
   目标:cname.vercel-dns.com
   代理状态:仅 DNS(关闭橙色云朵,避免和 Vercel CDN 冲突)
   ```
2. 在 Vercel 点 **"Refresh"** → 自动签 SSL → 完成

**情况 B:域名在阿里云/腾讯云(国内注册商)**

1. 添加 CNAME 记录:
   ```
   主机记录:@
   记录类型:CNAME
   记录值:cname.vercel-dns.com
   ```
2. 同样在 Vercel 刷新验证

### 3.3 域名跳转

建议同时绑 `sanzhanpeijiang.com` 和 `www.sanzhanpeijiang.com`,在 Vercel 把 `www` 跳到根域名(反之亦然)。

## 4. SSL / HTTPS

**Vercel 自动签发和续期 Let's Encrypt 证书**,无需任何操作。

- 首次部署:约 1-2 分钟内自动签发
- 续期:Let's Encrypt 证书 90 天到期前自动续
- TLS 1.3 + HTTP/2 默认开启
- 强制 HTTPS:Vercel 默认 301 跳 HTTP → HTTPS

**国内注册商(阿里云/腾讯云)需要先做 ICP 备案**才能用国内 CDN。Vercel 节点在境外,不需要备案,但访问速度会受影响(可用 Cloudflare 优选 IP 缓解)。

### 4.1 国内加速方案(可选)

如果目标用户主要在国内,可考虑:

| 方案 | 优点 | 缺点 |
| --- | --- | --- |
| **Vercel + Cloudflare** | 免费,海外回源 | 国内访问偶尔慢 |
| **Netlify + 国内备案** | 稳定 | 需要备案 |
| **腾讯云 Webify** | 国内节点,免运维 | 需备案 + 适配 Next.js |
| **自建 Next.js + Nginx** | 完全可控 | 运维成本高 |

MVP 阶段先用 Vercel + Cloudflare,如果国内用户占比超过 50% 再考虑迁移。

## 5. 环境变量

MVP 阶段**无外部依赖**,无需环境变量。

M2 之后可能需要:

| 变量名 | 用途 | 示例值 |
| --- | --- | --- |
| `NEXT_PUBLIC_SITE_URL` | 站点 URL(用于 sitemap、canonical) | `https://sanzhanpeijiang.com` |
| `VERCEL_ANALYTICS_ID` | Vercel Analytics ID(可选) | 自动注入 |
| `BING_WEBMASTER_TOKEN` | 必应站长 API(可选) | `<token>` |
| `BAIDU_PUSH_TOKEN` | 百度站长 API(可选) | `<token>` |

### 在 Vercel 配置环境变量

1. 项目 → **Settings → Environment Variables**
2. 添加 Key/Value
3. 选择生效环境:Production / Preview / Development
4. 保存后**重新部署**才会生效

## 6. CI / CD

详见 [`.github/workflows/ci.yml`](../.github/workflows/ci.yml),关键点:

- **触发**:`push` 到 `main` 和所有 `pull_request`
- **Jobs**:lint(ESLint)+ format-check(Prettier)+ typecheck(`tsc`)+ test(Vitest)
- **Node 20 + pnpm 缓存**(actions/setup-node 自带)

### 部署触发

Vercel 通过 GitHub App 监听 push:

| 事件 | 部署目标 |
| --- | --- |
| `push` 到 `main` | Production(线上正式环境) |
| `push` 到其他分支 | Preview URL(预览,带过期时间) |
| PR | 提交时自动部署一个 Preview,评论里会显示链接 |

### 部署超时

Vercel 默认部署超时 45 秒。如果 `pnpm build` 超过这个时间:

1. 在 Vercel 项目 → **Settings → Functions** 把 timeout 调到 60s 或 90s
2. 或者优化构建(见 §10 性能优化)

## 7. SEO 与站长平台

### 7.1 Sitemap

Next.js App Router 提供了 `sitemap.ts` 约定。在 `src/app/sitemap.ts` 写:

```typescript
import type { MetadataRoute } from 'next'
import { loadGenerals, loadSkills, loadLineups } from '@/lib/data/loader'

export default async function sitemap(): Promise<MetadataRoute.Sitemap> {
  const baseUrl = process.env.NEXT_PUBLIC_SITE_URL || 'https://sanzhanpeijiang.com'

  // 静态页面
  const staticPages = [
    '',
    '/generals',
    '/skills',
    '/sandbox',
    '/lineups',
    '/search',
    '/battle',
    '/traits',
    '/patches',
  ].map((route) => ({
    url: `${baseUrl}${route}`,
    lastModified: new Date(),
    changeFrequency: 'weekly' as const,
    priority: route === '' ? 1.0 : 0.8,
  }))

  // 动态页面
  const generals = loadGenerals()
  const skills = loadSkills()
  const lineups = loadLineups()

  const generalPages = generals.map((g) => ({
    url: `${baseUrl}/generals/${g.id}`,
    lastModified: new Date(),
    changeFrequency: 'weekly' as const,
    priority: 0.7,
  }))

  const skillPages = skills.map((s) => ({
    url: `${baseUrl}/skills/${s.id}`,
    lastModified: new Date(),
    changeFrequency: 'weekly' as const,
    priority: 0.7,
  }))

  const lineupPages = lineups.map((l) => ({
    url: `${baseUrl}/lineups/${l.id}`,
    lastModified: new Date(),
    changeFrequency: 'weekly' as const,
    priority: 0.7,
  }))

  return [...staticPages, ...generalPages, ...skillPages, ...lineupPages]
}
```

部署后访问 `https://sanzhanpeijiang.com/sitemap.xml` 应能看到 XML。

### 7.2 robots.txt

`src/app/robots.ts`:

```typescript
import type { MetadataRoute } from 'next'

export default function robots(): MetadataRoute.Robots {
  return {
    rules: [
      { userAgent: '*', allow: '/', disallow: ['/api/'] },
    ],
    sitemap: 'https://sanzhanpeijiang.com/sitemap.xml',
  }
}
```

### 7.3 百度站长平台

1. 注册 [百度搜索资源平台](https://ziyuan.baidu.com/)
2. 添加站点(需要验证网站所有权:文件验证 / CNAME 验证 / HTML 标签验证)
3. **普通收录** → 提交 sitemap:
   - 填入 `https://sanzhanpeijiang.com/sitemap.xml`
4. (可选)使用**推送 API** 实时推送新链接:
   ```bash
   curl -H 'Content-Type: text/plain' --data-binary @urls.txt "http://data.zz.baidu.com/urls?site=https://sanzhanpeijiang.com&token=YOUR_TOKEN"
   ```
   M2 可在 Vercel 配 Cron Job 每天跑一次。

### 7.4 必应站长平台

1. 注册 [Bing Webmaster Tools](https://www.bing.com/webmasters)
2. 添加站点,验证(类似百度)
3. 提交 sitemap:`https://sanzhanpeijiang.com/sitemap.xml`

### 7.5 Google Search Console(可选)

如果也想被 Google 收录:

1. [Google Search Console](https://search.google.com/search-console) 添加站点
2. 提交 sitemap
3. 选 "URL Prefix" 验证方式最方便(DNS TXT)

### 7.6 meta 标签

每个详情页都应有:

```tsx
export async function generateMetadata({ params }: { params: { id: string } }) {
  const general = loadGenerals().find((g) => g.id === params.id)
  return {
    title: `${general?.name} - 三战配将助手`,
    description: `${general?.name}(${general?.camp}国${general?.quality}将)四维属性、自带战法、传承战法详情`,
  }
}
```

## 8. 监控与日志

### 8.1 Vercel Analytics(可选)

1. 项目 → **Analytics** → Enable
2. 会自动注入 `@vercel/analytics`(需要 `pnpm add @vercel/analytics`)
3. 在 `src/app/layout.tsx` 引入:
   ```tsx
   import { Analytics } from '@vercel/analytics/react'
   // ...
   <body>
     {children}
     <Analytics />
   </body>
   ```
4. 看 Dashboard 可知道:PV / UV / 跳出率 / 加载时间

### 8.2 Vercel Speed Insights(可选)

1. 项目 → **Speed Insights** → Enable
2. 安装:`pnpm add @vercel/speed-insights`
3. 在 `layout.tsx` 引入 `<SpeedInsights />`
4. 监控 Core Web Vitals(LCP / FID / CLS)

### 8.3 错误监控

MVP 阶段**不开外部错误监控**(Sentry 等)。

如果后续要接:

- **Sentry**:免费 5K events/月,够用
  ```bash
  pnpm add @sentry/nextjs
  ```
- 配置 `SENTRY_DSN` 环境变量

### 8.4 日志

Vercel 自带访问日志和函数日志:

- 项目 → **Logs** 看实时
- 项目 → **Settings → Logs** 配置保留期(默认 1 小时,Pro 7 天)

## 9. 回滚与降级

### 9.1 Vercel 一键回滚

1. 项目 → **Deployments**
2. 找到上一个稳定的部署
3. 点击右上角 **"..."** → **"Promote to Production"**
4. 立即生效(CDN 缓存会在几秒内更新)

### 9.2 数据库迁移失败(后续)

M2 之后如果加了数据库:

```bash
# Vercel Postgres / Supabase 都支持 migration 工具
pnpm prisma migrate deploy
```

如果 migration 失败:

1. Vercel 一键回滚(同 §9.1)
2. 修复 migration 文件
3. 重新部署

### 9.3 数据回滚

MVP 阶段数据在 Git 里,数据问题直接 `git revert` + push,CI 自动重部署。

## 10. 性能优化清单

### 10.1 Next.js 默认优化(已开启)

- ✅ 静态生成(SSG):每个详情页 build 时预渲染
- ✅ 增量静态再生成(ISR):数据更新后自动重新生成
- ✅ 图片优化:`<Image>` 组件自动转 WebP + 懒加载
- ✅ 字体优化:`next/font` 自动子集化
- ✅ 代码分割:每个路由独立打包

### 10.2 数据加载

- `data/*.json` 在 build 时通过 `fs.readFile` 读入,运行时无 IO
- 大列表页(武将 120+ 条)用 `generateStaticParams` 预渲染

### 10.3 缓存策略

- 静态资源(`/_next/static/*`):1 年 immutable
- 页面:CDN 默认 60 秒 SWR(stale-while-revalidate)
- API 路由:不缓存(或手动控制)

### 10.4 体积优化

```bash
# 分析 bundle 体积
pnpm build
# 查看 .next/analyze/ 报告
```

如有异常大块,考虑:
- 拆分 `echarts` → 按需引入
- 用 `dynamic()` 拆分客户端组件

### 10.5 国内访问优化

- 启用 Cloudflare(免费)
- 在 Cloudflare 选 "亚洲" 节点
- 国内访问会自动就近回 Vercel

## 11. 常见问题 FAQ

### Q1:部署后打开是空白页?

- 检查 `pnpm build` 是否有报错
- 查看 Vercel **Build Logs**
- 大概率是 `next/image` 用了未配置域名的图片源,需要 `next.config.mjs` 加 `images.domains`

### Q2:部署成功但 404?

- 检查 `src/app/xxx/page.tsx` 是否存在
- Next.js App Router 的路由是**文件夹名**作为 URL,不是文件名

### Q3:更新数据后线上没生效?

- Vercel 默认 ISR revalidate 是 60 秒,等 1 分钟
- 或者手动触发:**Deployments → Redeploy**

### Q4:域名解析了但访问不到?

- DNS 全球生效需要 24-48 小时
- 用 `nslookup your-domain.com` 验证
- 检查 Vercel SSL 状态(Settings → Domains)

### Q5:需要备案吗?

- 用 Vercel/Netlify 等境外服务 → **不需要**
- 域名注册商(阿里云/腾讯云)注册本身也不需要备案
- **只有把服务器放国内**才需要备案

### Q6:CDN 缓存怎么清?

- Vercel:**Deployments → Redeploy**(会清缓存重建)
- Cloudflare:Cache → Purge Cache → Purge Everything

### Q7:怎么看到真实的访问数据?

- Vercel Analytics(见 §8.1)
- 百度站长平台 → 流量与关键词
- 必应站长平台 → 报告

### Q8:部署失败了怎么排查?

1. 看 Vercel Build Logs 找第一个错误
2. 本地先跑 `pnpm build` 复现
3. 检查 `package.json` 是否有未声明的依赖
4. 检查 `engines.node` 是不是 20

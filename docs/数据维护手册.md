## 0. 数据架构总览

所有数据存放在项目根目录的 `data/` 下,每个实体一个 JSON 文件,文件内是数组。

| 文件 | 实体 | 条数(MVP) | 条数(全量目标) |
| --- | --- | --- | --- |
| `data/generals.json` | 武将 | 5(示例) | 120+ |
| `data/skills.json` | 战法 | 10(示例) | 200+ |
| `data/tactics.json` | 兵书 | 3(示例) | 10-20 |
| `data/lineups.json` | 阵容 | 1(示例) | 10-20 |
| `data/traits.json` | 特技 | 5(示例) | 30+ |
| `data/items.json` | 装备 | 0(MVP 弱化) | (后续) |
| `data/patches.json` | 版本特性 | 0 | 持续累积 |
| `data/sim-config.json` | 模拟配置 | 1(默认) | 1 |

类型定义在 [`src/types/data.ts`](../src/types/data.ts),运行期校验用 Zod schema(在 [`src/lib/data/schemas.ts`](../src/lib/data/schemas.ts))。**修改数据不需要改代码**,只要符合 schema 即可。

加载流程:
```
JSON 文件 → fs.readFile → JSON.parse → Zod.parse → TS 类型数据
```

## 1. 录入武将(General)

### 字段说明

| 字段 | 类型 | 必填 | 含义 | 例子 |
| --- | --- | --- | --- | --- |
| `id` | string | ✅ | 唯一 ID(小写英文+下划线) | `"zhuge_liang"` |
| `name` | string | ✅ | 中文名 | `"诸葛亮"` |
| `camp` | enum | ✅ | 阵营:`蜀`/`魏`/`吴`/`群`/`汉` | `"蜀"` |
| `quality` | enum | ✅ | 品质:`橙`/`紫`/`蓝`/`绿` | `"橙"` |
| `stats` | object | ✅ | 四维属性 | `{ "武力": 78, "智力": 156, "统率": 96, "速度": 48 }` |
| `stats.武力` | number | ✅ | 武力值(30-200) | `78` |
| `stats.智力` | number | ✅ | 智力值 | `156` |
| `stats.统率` | number | ✅ | 统率值 | `96` |
| `stats.速度` | number | ✅ | 速度值 | `48` |
| `cavalry` | enum | ✅ | 骑兵适性:`S`/`A`/`B`/`C` | `"S"` |
| `shield` | enum | ✅ | 盾兵适性 | `"A"` |
| `archer` | enum | ✅ | 弓兵适性 | `"S"` |
| `spear` | enum | ✅ | 枪兵适性 | `"A"` |
| `siege` | enum | ✅ | 器械适性 | `"C"` |
| `trait` | enum | ✅ | 兵种特性描述 | `"主动"` |
| `selfSkillId` | string | ✅ | 自带战法 ID(引用 Skill.id) | `"qixing"` |
| `inheritSkillId` | string\|null | ✅ | 传承战法 ID(可空) | `"kongcheng"` |
| `redLevel` | number | ✅ | 红度/进阶数(0-5) | `0` |
| `learnableFormationSkillIds` | string[] | ✅ | 可学的阵法战法 ID 列表(子集为 `Skill.subType="阵法"`) | `["bamen_jinsuo", "wufeng_zheng"]` |
| `tacticsOptions.major` | string[] | ✅ | 可选大兵书 ID 列表 | `["tactic_001", "tactic_002"]` |
| `tacticsOptions.minor` | string[] | ✅ | 可选小兵书 ID 列表 | `["tactic_101", "tactic_102"]` |
| `equippableTraitCount` | number | ✅ | 可装备的特性(特技)数量上限(1-3) | `1` |

### 完整示例

```json
{
  "id": "zhuge_liang",
  "name": "诸葛亮",
  "camp": "蜀",
  "quality": "橙",
  "stats": { "武力": 78, "智力": 156, "统率": 96, "速度": 48 },
  "cavalry": "S",
  "shield": "A",
  "archer": "S",
  "spear": "A",
  "siege": "C",
  "trait": "主动",
  "selfSkillId": "qixing",
  "inheritSkillId": "kongcheng",
  "redLevel": 0,
  "learnableFormationSkillIds": ["bamen_jinsuo", "wufeng_zheng"],
  "tacticsOptions": {
    "major": ["tactic_001", "tactic_002"],
    "minor": ["tactic_101", "tactic_102"]
  },
  "equippableTraitCount": 1
}
```

### 录入步骤

1. 打开 `data/generals.json`
2. 在数组末尾追加一个对象
3. `id` 严格按 [§9 ID 命名规范](#9-id-命名规范)
4. 数值类属性(武力/智力/统率/速度)范围 30-200,过高过低都要核对游戏内数据
5. `selfSkillId` 和 `inheritSkillId` 必须**先在 `data/skills.json` 存在**(可先录入战法,再录入武将,顺序灵活)
6. `learnableFormationSkillIds` 数组中的每个 ID 都必须**对应一个 `subType = "阵法"` 的战法**
7. 保存,跑 `pnpm typecheck` 验证 TS 类型,通过后跑 `pnpm test`(数据 schema 校验)

## 2. 录入战法(Skill)

### 字段说明

| 字段 | 类型 | 必填 | 含义 | 例子 |
| --- | --- | --- | --- | --- |
| `id` | string | ✅ | 唯一 ID | `"qixing"` |
| `name` | string | ✅ | 中文名 | `"七星阵"` |
| `subType` | enum | ✅ | **6 类之一** | `"主动"` |
| `quality` | enum | ✅ | 品质 | `"橙"` |
| `description` | string | ✅ | 效果描述 | `"对敌方群体造成谋略伤害..."` |
| `source` | string | ✅ | 哪个武将自带/传承 | `"诸葛亮自带"` |
| `sourceType` | enum | ✅ | `自带` / `传承` / `拆解` / `通用` | `"自带"` |
| `carrierIds` | string[] | ✅ | 可携带该战法的武将 ID 列表 | `["zhuge_liang"]` |
| `triggerRate` | number\|null | ✅ | 发动概率(主动/突击 0-1,其他可为 null) | `0.35` |
| `multiTarget` | boolean | ✅ | 是否多目标(影响多穿评分) | `false` |
| `startRound` | number | ✅ | 战法首次可能发动的回合(影响节奏评分) | `1` |

### 战法 6 类(`subType`)

| subType | 含义 | 例子 |
| --- | --- | --- |
| `主动` | 主动施放,需概率发动 | 太平道法、刮骨疗毒 |
| `被动` | 战斗中自动生效,大概率触发 | 八门金锁、士别三日 |
| `指挥` | 战斗开始前生效,持续全场 | 武峰阵、竭力佐谋 |
| `突击` | 普攻后概率触发 | 益其利词 |
| `阵法` | 阵型类战法,选一个为队伍生效 | 武峰阵、八门金锁阵、藤甲阵、锋矢阵 |
| `兵种` | 兵种专属战法,部分武将/兵种才可学 | 西凉铁骑、白马义从 |

### 完整示例

```json
{
  "id": "qixing",
  "name": "七星阵",
  "subType": "主动",
  "quality": "橙",
  "description": "对敌方群体造成谋略伤害,智力差影响伤害系数",
  "source": "诸葛亮自带",
  "sourceType": "自带",
  "carrierIds": ["zhuge_liang"],
  "triggerRate": 0.35,
  "multiTarget": false,
  "startRound": 1
}
```

### 录入步骤

1. 打开 `data/skills.json`
2. 数组末尾追加一个对象
3. `subType` 必须是 6 个值之一,游戏里模糊的(武峰阵"既阵法又指挥")按**官方分类**填,本仓库默认填 `"阵法"`
4. `triggerRate`:**主动/突击**类型必填 0-1 之间的小数(35% 写 `0.35`);**被动/指挥/阵法/兵种**可为 `null`
5. `multiTarget`:多目标战法(太平道法、破阵催坚)写 `true`;单体写 `false`
6. `startRound`:1 表示第 1 回合就可能发动;部分被动写 1 也可,具体看游戏内机制
7. 保存,跑 `pnpm typecheck` 验证

## 3. 录入兵书(Tactics)

> 兵书 = 武将身上可装备的**非战法系统**(跟特技一样,都是"装备品",**不是**战法 6 类之一)。
> F3 配将时与"战法"分组展示,跟特技一起放在"装备品"区。

### 字段说明

| 字段 | 类型 | 必填 | 含义 | 例子 |
| --- | --- | --- | --- | --- |
| `id` | string | ✅ | 唯一 ID | `"tactic_001"` |
| `name` | string | ✅ | 中文名(常用"大兵书·xxx"/"小兵书·xxx"格式) | `"大兵书·攻其不备"` |
| `slot` | enum | ✅ | `major`(大兵书)/ `minor`(小兵书) | `"major"` |
| `category` | enum | ✅ | 类型:`输出`/`防御`/`谋略`/`速度`/`特殊` | `"输出"` |
| `effect` | string | ✅ | 效果描述 | `"对处于控制状态的敌人伤害 +12%"` |
| `appliesTo` | enum | ✅ | 作用对象:`主将`/`副将`/`全队` | `"主将"` |

### 完整示例

```json
{
  "id": "tactic_001",
  "name": "大兵书·攻其不备",
  "slot": "major",
  "category": "输出",
  "effect": "对处于控制状态的敌人伤害 +12%",
  "appliesTo": "主将"
}
```

### 录入步骤

1. 打开 `data/tactics.json`
2. 数组末尾追加一个对象
3. `slot` 选 `major`(大兵书)或 `minor`(小兵书)
4. 三战的"大兵书"全队通用(可被任何武将选),"小兵书"主将专属
5. 每位武将配 1 大 + 1 小,3 人队伍共 6 本
6. 保存,跑 `pnpm typecheck`

## 4. 录入阵容(Lineup)

### 字段说明

| 字段 | 类型 | 必填 | 含义 | 例子 |
| --- | --- | --- | --- | --- |
| `id` | string | ✅ | 唯一 ID | `"wufeng_zhangjiao_v1"` |
| `name` | string | ✅ | 阵容名 | `"武峰张角盾"` |
| `tier` | enum | ✅ | 评级:`T0` / `T1` / `T2` / `T3` | `"T1"` |
| `tags` | string[] | ✅ | 标签(自由文本,如"打架"/"PVP"/"开荒") | `["打架", "PVP"]` |
| `generalIds` | string[] | ✅ | 3 个武将 ID(顺序:主将→副将→副将) | `["diaochan", "zhangjiao", "zuoci"]` |
| `generalRedLevels` | object | ✅ | 每个武将的红度(0-5) | `{ "diaochan": 0, "zhangjiao": 3, "zuoci": 0 }` |
| `formationSkillId` | string | ✅ | 阵法战法 ID(必须对应一个 `subType="阵法"` 的战法) | `"wufeng_zheng"` |
| `troop` | enum | ✅ | 主兵种:`cavalry`/`shield`/`archer`/`spear`/`siege` | `"shield"` |
| `skills.main` | object | ✅ | 主将战法(1 主 + 2 副) | `{ "diaochan": ["yiqilici", "bamen"] }` |
| `skills.vice` | object | ✅ | 副将战法(每人 2 个) | `{ "zhangjiao": ["taipingdaofa", "shibie"], "zuoci": ["jieli", "guagu"] }` |
| `tactics.major` | string[] | ✅ | 3 个大兵书 ID(对应 3 个武将) | `["tactic_001", "tactic_002", "tactic_003"]` |
| `tactics.minor` | string[] | ✅ | 3 个小兵书 ID(对应 3 个武将) | `["tactic_101", "tactic_102", "tactic_103"]` |
| `equippedTraitIds` | string[] | ✅ | 装备的特性 ID 列表(可空数组) | `["trait_tiangong"]` |
| `description` | string | ✅ | 阵容说明 | `"貂蝉主将分担伤害,张角核弹输出..."` |
| `counters` | string[] | ✅ | 克制谁(阵容名,自由文本) | `["核弹张飞弓", "传统父女枪"]` |
| `counteredBy` | string[] | ✅ | 被谁克制 | `["太尉盾", "北伐枪"]` |
| `ratings.output` | number | ✅ | 6 维评分(0-100) | `48` |
| `ratings.recover` | number | ✅ | 回复/治疗 | `47` |
| `ratings.multihit` | number | ✅ | 多穿/穿透 | `58` |
| `ratings.rhythm` | number | ✅ | 节奏/速度 | `39` |
| `ratings.coverage` | number | ✅ | 打击面 | `47` |
| `ratings.stability` | number | ✅ | 稳定性 | `57` |
| `ratings.total` | number | ✅ | 综合分(MVP 阶段手填,M2 自动算) | `50.7` |
| `tierByScore` | enum | ✅ | 根据 total 推算的评级 | `"T2"` |

### 完整示例

```json
{
  "id": "wufeng_zhangjiao_v1",
  "name": "武峰张角盾",
  "tier": "T1",
  "tags": ["打架", "PVP"],
  "generalIds": ["diaochan", "zhangjiao", "zuoci"],
  "generalRedLevels": { "diaochan": 0, "zhangjiao": 3, "zuoci": 0 },
  "formationSkillId": "wufeng_zheng",
  "troop": "shield",
  "skills": {
    "main": { "diaochan": ["yiqilici", "bamen"] },
    "vice": {
      "zhangjiao": ["taipingdaofa", "shibie"],
      "zuoci": ["jieli", "guagu"]
    }
  },
  "tactics": {
    "major": ["tactic_001", "tactic_002", "tactic_003"],
    "minor": ["tactic_101", "tactic_102", "tactic_103"]
  },
  "equippedTraitIds": ["trait_tiangong"],
  "description": "貂蝉主将分担伤害,张角核弹输出,左慈续航",
  "counters": ["核弹张飞弓", "传统父女枪"],
  "counteredBy": ["太尉盾", "北伐枪"],
  "ratings": {
    "output": 48,
    "recover": 47,
    "multihit": 58,
    "rhythm": 39,
    "coverage": 47,
    "stability": 57,
    "total": 50.7
  },
  "tierByScore": "T2"
}
```

### 录入步骤

1. 打开 `data/lineups.json`
2. 数组末尾追加一个对象
3. **录入前**确保 `generalIds` 中的所有武将、`skills` 中所有战法、`formationSkillId` 对应的阵法战法、`tactics` 中所有兵书、`equippedTraitIds` 中所有特技都已经在对应 JSON 文件中存在
4. `generalRedLevels` 的 key 必须**完全等于** `generalIds` 中的 ID(否则 TS 类型校验会失败)
5. `skills.main` 的 key 是主将 ID,`skills.vice` 的 keys 是 2 个副将 ID
6. `tactics.major` 和 `tactics.minor` 数组长度必须 = 3(对应 3 个武将)
7. `ratings.total` 是综合分,通常 = (output + recover + multihit + rhythm + coverage + stability) / 6,保留 1 位小数
8. `tierByScore` 根据 `total` 推算:
   - `total >= 55` → `T0`
   - `total >= 45` → `T1`
   - `total >= 35` → `T2`
   - 其他 → `T3`
9. 保存,跑 `pnpm typecheck` + `pnpm test` 验证引用一致性

## 5. 录入特技(Trait)

> 特技 = 装备附带的特性(MVP 不做完整装备库,只做这部分)。
> F8 已重命名为"特技库"。

### 字段说明

| 字段 | 类型 | 必填 | 含义 | 例子 |
| --- | --- | --- | --- | --- |
| `id` | string | ✅ | 唯一 ID(建议加 `trait_` 前缀) | `"trait_tiangong"` |
| `name` | string | ✅ | 中文名 | `"天公"` |
| `ownerGeneralId` | string\|null | ✅ | 专属武将 ID(通用特性写 `null`) | `"zhangjiao"` |
| `triggerCondition` | string | ✅ | 触发条件 | `"施放天雷时"` |
| `effect` | string | ✅ | 效果描述 | `"天雷伤害 +20%,且必定暴击"` |
| `source` | string | ✅ | 来源(哪个装备附带) | `"张角专属装备附带"` |
| `category` | enum | ✅ | 分类:`攻击`/`防御`/`谋略`/`速度`/`特殊` | `"特殊"` |

### 完整示例

专属特技(`ownerGeneralId` 不为 null):
```json
{
  "id": "trait_tiangong",
  "name": "天公",
  "ownerGeneralId": "zhangjiao",
  "triggerCondition": "施放天雷时",
  "effect": "天雷伤害 +20%,且必定暴击",
  "source": "张角专属装备附带",
  "category": "特殊"
}
```

通用特技(`ownerGeneralId` 为 null):
```json
{
  "id": "trait_crit_10",
  "name": "暴击强化",
  "ownerGeneralId": null,
  "triggerCondition": "无",
  "effect": "暴击率 +10%",
  "source": "通用装备附带",
  "category": "攻击"
}
```

### 录入步骤

1. 打开 `data/traits.json`
2. 数组末尾追加一个对象
3. **通用特性**(`ownerGeneralId = null`):谁都能带,占大多数
4. **专属特性**(`ownerGeneralId = "武将ID"`):只能给该武将,`ownerGeneralId` 必须存在于 `data/generals.json`
5. `category` 影响 F8 特技库的筛选分类
6. 保存,跑 `pnpm typecheck`

## 6. 录入装备(Item)(MVP 弱化)

> MVP 阶段**不主动建装备库**,只在 F8 特技库里展示"哪些装备附带了哪些特性"。装备本身是附属。
> 数据上仍保留 Item 表的 schema 供后续扩展。

### 字段说明

| 字段 | 类型 | 必填 | 含义 |
| --- | --- | --- | --- |
| `id` | string | ✅ | 唯一 ID(建议加 `equip_` 前缀) |
| `name` | string | ✅ | 装备名 |
| `category` | enum | ✅ | `装备` |
| `quality` | enum | ✅ | 品质 |
| `slot` | enum | ✅ | 部位:`头`/`甲`/`马`/`盾`/`武器` |
| `baseStats` | object | ✅ | 基础属性(自由 key,如 `{"统率": 15, "智力": 10}`) |
| `traits` | array | ✅ | 附带特性(指向 Trait) |
| `traits[].traitId` | string | ✅ | 特性 ID(引用 Trait.id) |
| `traits[].value` | number | ✅ | 数值(如 0.10 表示 +10%) |
| `source` | string | ✅ | 来源(锻造/副本掉率等) |
| `compatibleGeneralIds` | string[] | ✅ | 限定武将(空数组表示通用) |

### 完整示例

```json
{
  "id": "equip_helmet_s",
  "name": "寒铁盔",
  "category": "装备",
  "quality": "橙",
  "slot": "头",
  "baseStats": { "统率": 15, "智力": 10 },
  "traits": [
    { "traitId": "trait_crit_10", "value": 0.10 }
  ],
  "source": "锻造 / 副本掉率",
  "compatibleGeneralIds": []
}
```

### 录入步骤

1. 打开 `data/items.json`
2. **MVP 阶段建议不录**,只在 F8 通过 Trait 反向展示"哪些装备附带了该特性"
3. 如果要录,`traits[].traitId` 必须存在于 `data/traits.json`

## 7. 录入版本特性(Patch)

### 字段说明

| 字段 | 类型 | 必填 | 含义 | 例子 |
| --- | --- | --- | --- | --- |
| `id` | string | ✅ | 唯一 ID(建议 `patch_YYYY_MM` 格式) | `"patch_2026_05"` |
| `version` | string | ✅ | 版本号 | `"v1.5.2"` |
| `date` | string(YYYY-MM-DD) | ✅ | 更新日期 | `"2026-05-10"` |
| `type` | enum | ✅ | 类型:`武将调整` / `战法调整` / `新增武将` / `新增战法` / `赛季机制` | `"武将调整"` |
| `summary` | string | ✅ | 一句话总结 | `"诸葛亮自带战法伤害系数下调 8%"` |
| `details` | string | ✅ | 详细描述 | (长文本) |
| `affectedIds` | string[] | ✅ | 受影响的武将/战法 ID 列表 | `["zhuge_liang"]` |

### 完整示例

```json
{
  "id": "patch_2026_05",
  "version": "v1.5.2",
  "date": "2026-05-10",
  "type": "武将调整",
  "summary": "诸葛亮自带战法伤害系数下调 8%",
  "details": "诸葛亮自带战法【七星阵】的伤害系数从 1.0 下调至 0.92,智力差加成不变。",
  "affectedIds": ["zhuge_liang", "qixing"]
}
```

### 录入步骤

1. 打开 `data/patches.json`
2. 数组末尾追加一个对象
3. 三战官方每次大版本更新后录入一条
4. `affectedIds` 中可以是武将 ID 或战法 ID,混在一起
5. F9 页面按 `date` 倒序展示,玩家可看到近 3 个月的调整记录

## 8. 修改模拟配置(SimConfig)

> SimConfig 是 F7 模拟交战用的系数表,只一个对象(不是数组)。

### 字段说明

| 字段 | 类型 | 含义 | 例子 |
| --- | --- | --- | --- |
| `iterations` | number | 模拟次数(MVP 默认 1000) | `1000` |
| `triggerRate.主动` | number | 主动战法发动概率 | `0.35` |
| `triggerRate.被动` | number | 被动战法发动概率 | `1.0` |
| `triggerRate.突击` | number | 突击战法发动概率 | `0.35` |
| `triggerRate.指挥` | number | 指挥战法发动概率 | `1.0` |
| `troopCounter.骑-盾` | number | 骑兵打盾兵伤害系数 | `1.2` |
| `troopCounter.盾-弓` | number | 盾兵打弓兵 | `1.2` |
| `troopCounter.弓-枪` | number | 弓兵打枪兵 | `1.2` |
| `troopCounter.枪-骑` | number | 枪兵打骑兵 | `1.2` |
| `troopCounter.盾-骑` | number | 盾兵被骑兵打(克) | `0.8` |
| `troopCounter.弓-盾` | number | 弓兵被盾兵打(克) | `0.8` |
| `troopCounter.枪-弓` | number | 枪兵被弓兵打(克) | `0.8` |
| `troopCounter.骑-枪` | number | 骑兵被枪兵打(克) | `0.8` |
| `troopCounter.same` | number | 同兵种 | `1.0` |
| `campBonus` | number | 同阵营加成 | `1.1` |

### 完整示例

```json
{
  "iterations": 1000,
  "triggerRate": { "主动": 0.35, "被动": 1.0, "突击": 0.35, "指挥": 1.0 },
  "troopCounter": {
    "骑-盾": 1.2, "盾-弓": 1.2, "弓-枪": 1.2, "枪-骑": 1.2,
    "盾-骑": 0.8, "弓-盾": 0.8, "枪-弓": 0.8, "骑-枪": 0.8,
    "same": 1.0
  },
  "campBonus": 1.1
}
```

### 修改步骤

1. 打开 `data/sim-config.json`
2. 直接修改数值(整个文件只有一个对象,不是数组)
3. `iterations` 越大越准但越慢,推荐 1000-5000
4. `troopCounter` 的 9 个 key 必须**全部存在**(被克制、被克、同兵种)
5. 保存,跑 `pnpm typecheck`

## 10. 引用一致性自检

数据文件之间存在大量 ID 引用关系,录入/修改时必须自检。CI 会跑自动化校验,本地也可以手动跑:

```bash
pnpm test
```

### 引用关系图

```
General.selfSkillId ──────────→ Skill.id
General.inheritSkillId ───────→ Skill.id
General.learnableFormationSkillIds[*] ─→ Skill.id (其中 subType="阵法")
General.tacticsOptions.major[*] ─→ Tactics.id
General.tacticsOptions.minor[*] ─→ Tactics.id

Skill.carrierIds[*] ───────→ General.id

Lineup.generalIds[*] ──────→ General.id
Lineup.generalRedLevels[*] ──→ General.id (key 集合相等)
Lineup.formationSkillId ──────→ Skill.id (其中 subType="阵法")
Lineup.skills.main[*] ────────→ Skill.id
Lineup.skills.vice[*][*] ─────→ Skill.id
Lineup.tactics.major[*] ──────→ Tactics.id
Lineup.tactics.minor[*] ──────→ Tactics.id
Lineup.equippedTraitIds[*] ───→ Trait.id

Trait.ownerGeneralId ────────→ General.id (可空)

Item.traits[*].traitId ──────→ Trait.id

Patch.affectedIds[*] ────────→ General.id 或 Skill.id
```

### 自检清单

录入完一条数据后,逐项核对:

- [ ] 武将的 `selfSkillId` / `inheritSkillId` 在 `skills.json` 存在
- [ ] 武将的 `learnableFormationSkillIds` 中每个 ID 对应一个 `subType="阵法"` 的战法
- [ ] 战法的 `carrierIds` 中每个 ID 在 `generals.json` 存在
- [ ] 阵容的 `generalIds` 中每个 ID 在 `generals.json` 存在
- [ ] 阵容的 `generalRedLevels` 的 key 集合 = `generalIds`
- [ ] 阵容的 `formationSkillId` 在 `skills.json` 存在且 `subType="阵法"`
- [ ] 阵容的 `skills.main` 和 `skills.vice` 中所有战法 ID 在 `skills.json` 存在
- [ ] 阵容的 `tactics.major`/`minor` 中所有 ID 在 `tactics.json` 存在
- [ ] 阵容的 `equippedTraitIds` 中所有 ID 在 `traits.json` 存在
- [ ] 特技的 `ownerGeneralId`(若非 null)在 `generals.json` 存在
- [ ] 装备的 `traits[].traitId` 在 `traits.json` 存在
- [ ] 版本特性的 `affectedIds` 至少有一条对应到现有 ID

CI 会在 `pnpm test` 阶段自动跑校验,所有引用关系不通过会导致 build 失败。

## 11. 提交流程

1. **本地编辑** — 在 `data/*.json` 中修改/新增
2. **本地校验**
   ```bash
   pnpm typecheck  # TS 类型校验
   pnpm test       # Zod schema + 引用一致性
   pnpm format     # 自动格式化
   pnpm lint       # 代码规范(虽然改 JSON 不会触发 lint,但保险起见跑一遍)
   ```
3. **commit** — 按规范写 commit message(参见 README.md 贡献指南)
   ```bash
   git add data/generals.json
   git commit -m "data: 录入张角、于吉、左慈 3 个蜀国橙将"
   ```
4. **push 并开 PR**
   ```bash
   git push origin feat/add-shu-generals
   gh pr create --title "data: 录入张角、于吉、左慈" --body "..."
   ```
5. **等 CI 通过** — lint + format-check + typecheck + test
6. **等 review** — 数据需要至少 1 个 reviewer 确认数值无问题
7. **合并** — squash merge 即可,合并后自动部署到预览环境

## 12. 常见问题 FAQ

### Q1:怎么加新战法类型(subType)?

战法严格分 6 类(`主动`/`被动`/`指挥`/`突击`/`阵法`/`兵种`),**不允许新增**。
如果游戏官方新增了第 7 类,需要:

1. 修改 `src/types/data.ts` 中 `Skill.subType` 的联合类型
2. 修改 `src/lib/data/schemas.ts` 中 `SkillSchema` 的 zod enum
3. 在 PRD 中记录这次扩展
4. 在 commit message 中说明 "feat: extend Skill.subType to 7 classes (xxx)"

### Q2:武将 ID 用拼音还是用数字?

用**拼音**(`zhuge_liang`)。原因:

- 可读性强,新人接手能直接读懂
- URL 友好(`/generals/zhuge_liang` 比 `/generals/1` SEO 更好)
- 跨语言兼容(英文环境也能识别)

避免使用 `g001` 这类无意义 ID,虽然对程序来说更"干净",但牺牲可读性。

### Q3:同一个战法有多个传承将怎么办?

在 `Skill.carrierIds` 数组中写所有可携带该战法的武将 ID(都来自 `generals.json`):

```json
{
  "id": "hengsao_qianjun",
  "name": "横扫千军",
  "carrierIds": ["zhao_yun", "guan_yu"]  // 2 个传承将
}
```

不在 `carrierIds` 里的武将不能配该战法(前端会过滤)。

### Q4:数字属性(武力/智力等)精度怎么定?

游戏内数据保留**整数**(无小数),如武力 78 写 `78` 不写 `78.0`。

### Q5:triggerRate 写 35% 还是 0.35?

**写 0.35**(小数表示概率,0-1 范围)。前端展示时再乘 100 加百分号。

### Q6:startRound 怎么填?

大多数战法都填 `1`(第 1 回合就可能发动)。
部分被动/指挥虽然战斗开始就生效,但代码里仍按 `1` 处理;
部分"延迟生效"战法(少见)按游戏内机制写 2/3 等。

### Q7:阵容的 ratings 怎么算?

MVP 阶段**手填**。计算公式(参考):

```
total = (output + recover + multihit + rhythm + coverage + stability) / 6
tierByScore:
  total >= 55 → T0
  total >= 45 → T1
  total >= 35 → T2
  其他         T3
```

M2 阶段会改成 F7 自动算(MVP 阶段先手填占位)。

### Q8:特技的 ownerGeneralId 为 null 时,F3 怎么过滤?

前端逻辑:

- `ownerGeneralId = null` → 所有武将都能装备(通用)
- `ownerGeneralId = "xxx"` → 只在阵容的"特技选择"里出现,且该武将必须在阵容中

### Q9:数据来源声明写在哪?

每个数据文件的**最后一条**可以加注释字段 `__note`(虽然 JSON 不支持注释,但可以在文件外另写),或者在 README 顶部声明:

> 数据来源:[gamekee](https://www.gamekee.com/)、[配将宝](https://www.peijiangbao.com/) 等公开资料,经人工核对整理,以游戏内为准。

### Q10:发现数据错误怎么办?

1. 站内"报错"按钮(F8/F9 页面底部)— 玩家可一键提问题
2. 直接在 GitHub Issues 开 issue,标签 `data-bug`
3. 自行修改并提 PR(请附数据来源截图)
